# Production Deployment Guide

This guide deploys the Internship Logging and Management System with Django 5, PostgreSQL, GitHub, Render, Gunicorn, Whitenoise, and environment variables.

## 1. Project Analysis

The project already follows Django's MTV structure:

- `core/`: project settings, root URLs, WSGI and ASGI entry points.
- `accounts/`: users, roles, student profiles, companies, departments, academic years, placements, dashboards.
- `logbook/`: daily logs, weekly reports, attendance, feedback, notifications.
- `evaluation/`: evaluations and university supervisor visit schedules.
- `templates/`: shared base layout and error pages.
- `static/`: project static assets.

Deployment issues fixed:

- `DEBUG` is now controlled by environment variables and defaults to `False`.
- `SECRET_KEY` is no longer hardcoded.
- `DATABASE_URL` is parsed with `dj-database-url`.
- PostgreSQL is supported through `psycopg`.
- Static files are served by Whitenoise.
- Gunicorn is included for production WSGI serving.
- Secure cookies, HTTPS redirects, HSTS, clickjacking protection, and content-type sniffing protection are configured.
- `404.html` and `500.html` are present.

## 2. Environment Variables

Copy `.env.example` to `.env` for local production-style testing:

```bash
cp .env.example .env
```

For local development, use values like:

```env
SECRET_KEY=local-long-secret-key-change-this-before-production
DEBUG=True
DATABASE_URL=
ALLOWED_HOSTS=localhost,127.0.0.1
CSRF_TRUSTED_ORIGINS=http://localhost,http://127.0.0.1
SECURE_SSL_REDIRECT=False
SESSION_COOKIE_SECURE=False
CSRF_COOKIE_SECURE=False
```

For Render, set:

```env
SECRET_KEY=<generated by Render>
DEBUG=False
DATABASE_URL=<provided by Render PostgreSQL>
ALLOWED_HOSTS=your-app-name.onrender.com
CSRF_TRUSTED_ORIGINS=https://your-app-name.onrender.com
SECURE_SSL_REDIRECT=True
SESSION_COOKIE_SECURE=True
CSRF_COOKIE_SECURE=True
```

The project uses `python-decouple`:

```python
from decouple import Csv, config

SECRET_KEY = config('SECRET_KEY')
DEBUG = config('DEBUG', default=False, cast=bool)
ALLOWED_HOSTS = config('ALLOWED_HOSTS', default='localhost,127.0.0.1', cast=Csv())
```

`config()` reads from environment variables first and then from `.env` if present.

## 3. PostgreSQL Configuration

The database is configured in `core/settings.py`:

```python
DATABASE_URL = config('DATABASE_URL', default='')
if DATABASE_URL:
    DATABASES = {
        'default': dj_database_url.parse(
            DATABASE_URL,
            conn_max_age=600,
            conn_health_checks=True,
            ssl_require=not DEBUG,
        )
    }
else:
    DATABASES = {
        'default': {
            'ENGINE': 'django.db.backends.sqlite3',
            'NAME': BASE_DIR / 'db.sqlite3',
        }
    }
```

Explanation:

- `DATABASE_URL`: one connection string containing database name, user, password, host, and port.
- `dj_database_url.parse()`: converts the URL into Django's `DATABASES` dictionary.
- `conn_max_age=600`: keeps database connections open for reuse.
- `conn_health_checks=True`: checks reused connections before queries.
- `ssl_require=not DEBUG`: requires SSL in production.
- SQLite fallback exists only for local development when `DATABASE_URL` is empty.

Migration commands:

```bash
python manage.py makemigrations
python manage.py migrate
```

On Render, migrations run automatically because `render.yaml` includes `python manage.py migrate` in the build command.

## 4. Static Files and Whitenoise

Installed package:

```txt
whitenoise==6.9.0
```

Middleware order:

```python
MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',
    'whitenoise.middleware.WhiteNoiseMiddleware',
    ...
]
```

Static settings:

```python
STATIC_URL = '/static/'
STATIC_ROOT = BASE_DIR / 'staticfiles'
STATICFILES_DIRS = [BASE_DIR / 'static'] if (BASE_DIR / 'static').exists() else []

STORAGES = {
    'default': {
        'BACKEND': 'django.core.files.storage.FileSystemStorage',
    },
    'staticfiles': {
        'BACKEND': 'whitenoise.storage.CompressedManifestStaticFilesStorage',
    },
}
```

Explanation:

- `STATIC_URL`: URL prefix used in HTML.
- `STATIC_ROOT`: folder where `collectstatic` gathers files.
- `STATICFILES_DIRS`: local project static folders.
- `CompressedManifestStaticFilesStorage`: compresses files and adds hashed names for browser caching.

Command:

```bash
python manage.py collectstatic --noinput
```

## 7. Render Blueprint

`render.yaml` defines one PostgreSQL database and one web service.

```yaml
databases:
  - name: internship-system-db
    plan: free
    databaseName: internship_system
    user: internship_user

services:
  - type: web
    name: internship-system
    runtime: python
    plan: free
    buildCommand: "pip install -r requirements.txt && python manage.py collectstatic --noinput && python manage.py migrate"
    startCommand: "gunicorn core.wsgi:application"
```

Explanation:

- `databases`: creates PostgreSQL.
- `services`: creates the Django web app.
- `runtime: python`: tells Render to use Python.
- `buildCommand`: installs dependencies, collects static files, applies migrations.
- `startCommand`: starts Django through Gunicorn.
- `DATABASE_URL`: pulled from the Render PostgreSQL connection string.

Important: change `internship-system.onrender.com` in `render.yaml` to your actual Render service hostname after creation if Render gives you a different name.

## 10. Security Settings

Production security code in `core/settings.py`:

```python
SECURE_PROXY_SSL_HEADER = ('HTTP_X_FORWARDED_PROTO', 'https')
SECURE_SSL_REDIRECT = config('SECURE_SSL_REDIRECT', default=not DEBUG, cast=bool)
SESSION_COOKIE_SECURE = config('SESSION_COOKIE_SECURE', default=not DEBUG, cast=bool)
CSRF_COOKIE_SECURE = config('CSRF_COOKIE_SECURE', default=not DEBUG, cast=bool)
SESSION_COOKIE_HTTPONLY = True
CSRF_COOKIE_HTTPONLY = False
SESSION_COOKIE_SAMESITE = 'Lax'
CSRF_COOKIE_SAMESITE = 'Lax'
SECURE_HSTS_SECONDS = config('SECURE_HSTS_SECONDS', default=31536000 if not DEBUG else 0, cast=int)
SECURE_HSTS_INCLUDE_SUBDOMAINS = config('SECURE_HSTS_INCLUDE_SUBDOMAINS', default=not DEBUG, cast=bool)
SECURE_HSTS_PRELOAD = config('SECURE_HSTS_PRELOAD', default=not DEBUG, cast=bool)
SECURE_CONTENT_TYPE_NOSNIFF = True
X_FRAME_OPTIONS = 'DENY'
SECURE_REFERRER_POLICY = 'same-origin'
```

What this protects:

- HTTPS support through Render proxy headers.
- Secure session and CSRF cookies.
- XSS and MIME-sniffing protection.
- Clickjacking protection.
- HSTS for strict HTTPS use.
- CSRF protection through Django middleware and trusted origins.

## 12. Performance Optimization

Already implemented:

- Whitenoise static compression.
- Hashed static files for browser cache busting.
- Database connection reuse with `conn_max_age=600`.
- Pagination on list pages.
- `select_related()` in several views for related user, placement, and company data.

Recommended next improvements:

- Add `select_related('student', 'company', 'department')` to every placement list.
- Add `prefetch_related()` where a page displays many logs, reports, or evaluations per student.
- Add indexes for frequent filters such as `status`, `date`, `student`, and `placement`.
- Use Redis caching for expensive dashboards if traffic grows.
- Move uploaded media to cloud storage such as Cloudinary, S3, or Render Disk for persistent production uploads.

## 14. Common Render Errors and Fixes

### DisallowedHost

Cause: your Render hostname is missing from `ALLOWED_HOSTS`.

Fix:

```env
ALLOWED_HOSTS=your-app-name.onrender.com
```

### CSRF verification failed

Cause: Render URL is missing from `CSRF_TRUSTED_ORIGINS`.

Fix:

```env
CSRF_TRUSTED_ORIGINS=https://your-app-name.onrender.com
```

### Static files not loading

Causes:

- `collectstatic` did not run.
- Whitenoise middleware is missing.
- Static storage is misconfigured.

Fix:

```bash
python manage.py collectstatic --noinput
```

Confirm `WhiteNoiseMiddleware` appears immediately after `SecurityMiddleware`.

### Database connection error

Cause: `DATABASE_URL` is missing or the database is not linked.

Fix: in Render, link the web service to the PostgreSQL database and confirm `DATABASE_URL` is present.

### No such table

Cause: migrations did not run.

Fix:

```bash
python manage.py migrate
```

### Application exited early

Cause: wrong Gunicorn module path.

Fix for this project:

```bash
gunicorn core.wsgi:application
```

### SECRET_KEY missing

Cause: `SECRET_KEY` environment variable was not set.

Fix: add `SECRET_KEY` in Render environment variables or let `render.yaml` generate it.
